# 产品需求文档 (PRD)：【真面 AI】(MockMaster)

## 一、 产品概述与核心价值

**产品愿景**：打破“面试黑盒”，为顶尖本科生提供媲美真实大厂业务一线的 AI 模拟面试与职业陪跑服务。
**核心价值**：通过结构化的知识库负载（RAG）和多模态 AI 技术，提供极具特异性、压力感和专业度的面试对练，拒绝“背诵式”辅导，培养用户的临场应变与真实问题解决能力。

## 二、 目标用户与痛点分析（补充）

**核心垂直受众**：
聚焦于泛互联网/AI方向（如大模型产品经理、NLP算法/评测实习生）**以及**顶尖高校学术保研（如计算语言学、计算机科学等方向）的硬核本科生。

**用户真实痛点（包含您已指出的并补充）：**

1. **找不到资深对练资源**：难以随时联系到大厂在职专家或目标院校的导师/学长进行高频次、高质量的 Mock。
2. **反馈缺乏可执行性**：市面上的工具给出的建议过于宽泛（如“不够自信”、“逻辑不清”），缺乏针对具体业务场景（如“这个特征工程你为什么不用X方法”或“你项目里的某个框架选型有漏洞”）的硬核反馈。
3. **【补充】过度依赖逐字稿，缺乏底层逻辑**：竞品多提供“完美参考答案”，导致学生像背书一样面试，一旦遇到面试官的压力追问（Follow-up）瞬间崩溃。
4. **【补充】不了解面试官的真实考察意图**：学生往往只回答表面问题，不知道 HR 或业务面背后实际在考察什么维度（如：抗压能力、跨部门协作、科研素养）。

## 三、 产品形态、交互与技术栈建议

为了实现上述目标，建议产品采用 **Web 端为主，小程序为辅** 的形态。面试场景需要安静的环境和专注的交互，Web 端更符合真实视频面试的物理场景。

* **产品形态**：沉浸式 Web 平台（带摄像头、麦克风权限调用）。
* **交互设计**：极简、专业的“暗黑/冷色调”拟真办公风格。主打“无感交互”，面试中极少需要鼠标点击，全程语音驱动。
* **技术栈与 AI 工具推荐**：
* **前端**：React.js + WebRTC（保障音视频低延迟传输）。
* **后端**：Python + FastAPI（适合并发和 AI 模型调用）。
* **核心 LLM**：建议接入推理能力极强的模型（如 GPT-4o 或 国内的 DeepSeek-V3/Doubao-pro），用于深挖简历逻辑。
* **RAG（检索增强生成）**：Milvus/Faiss 向量数据库，用于挂载您提供的 HR 手册、大厂面经和保研经验，确保提问的专业性。
* **音视频/数字人**：
* **语音合成 (TTS)**：火山引擎 (Volcengine) 或 Azure 的高拟真情感语音（带呼吸声、停顿感，模拟真实面试官）。
* **数字人引擎**：若预算充足可接入 HeyGen API 或火山引擎 2D 数字人，若为敏捷 MVP 版本，可用“动态声纹波形+面试官虚拟头像”代替实体数字人，降低延迟。

## 四、 核心功能模块与用户使用路径

产品的核心流程分为 **4 个递进阶段**：输入与匹配 -> 沉浸式准备 -> 真实模拟赛场 -> 深度复盘。

### 阶段一：建档与匹配 (Onboarding & Matching)

1. **简历解析上传**：
* 用户上传 PDF 简历（或选用平台模板）。AI 自动提取关键实体（教育背景、实习经历、项目经历）。


2. **岗位/方向选择**：
* 用户上传目标岗位的 JD（职位描述）截图，或输入保研的目标院校及导师研究方向。
* **【简短职业测试】**：若用户无明确目标，系统弹出 3-5 道情景选择题（例如偏好从 0 到 1 搭建框架还是从 1 到 100 优化指标，偏好高薪高压还是相对自由），结合简历，推荐 3 个最匹配的高潜力岗位/研究方向。


3. **简历“毒舌”诊断与修改**：
* AI 结合上传的 JD 和知识库，指出简历中的“致命伤”（如：数据不量化、项目职责不清晰、技术栈与岗位不匹配），并提供前后对比的修改建议。



### 阶段二：面试准备舱 (Preparation Mode) —— 核心亮点

此阶段主打“框架性启发”而非“填鸭式喂答案”。

1. **AI 生成三维题库**：
结合简历、JD、大厂面经知识库，生成三版难度递进的题库：
* **Level 1：常规基础题**（自我介绍、基础专业概念，如算法基础或产品常识）。
* **Level 2：简历深挖题**（特异性极高。例如提取简历中“做过适老化健康应用”或“大模型评测”，追问：“你在设计大模型评测指标时，如何解决长文本幻觉的评测量化问题？”）。
* **Level 3：压力/前沿挑战题**（情景假设、跨部门冲突、最新学术论文/行业热点）。


2. **要点提示式对练（语音+文字）**：
* 用户看着题目，点击录音开始自我作答。
* **【关键设计：要点提示】**：界面**绝对不显示**完整的标准答案。如果用户卡壳，点击“提示”，AI 仅弹出 3-5 个骨架关键词（例如：*STAR 法则、强调你的数据清洗方案、提及用户隐私合规*）。
* **即时反馈**：录音结束后，AI 输出用户的语音转文字稿，并从“逻辑结构、专业度、契合度”三个维度给出个性化修改建议。



### 阶段三：全真模拟面试赛场 (Real Mock Mode)

完全模拟真实的飞书/腾讯会议面试场景。

1. **环境配置**：强制开启摄像头（视频录制）与麦克风。
2. **数字人登场**：AI 数字人（如设定为“字节跳动资深技术专家”或“某高校博导”）进行语音互动。
3. **高压追问机制**：
* AI 面试官不仅会问预设题库，还会根据用户上一句话的回答，实时生成 **Follow-up 追问**（例如：用户回答“我用了某某策略提升了留存”，AI 立刻打断追问“但这个策略会带来服务器成本上升，你怎么权衡？”）。
* 加入随机的“压力测试”（如数字人表现出轻微的不耐烦，或质疑用户的某项数据）。


4. **过程全记录**：后台同步录制用户的视频表现（表情、眼神、微动作）和音频。

### 阶段四：赛后报告与职业规划 (Post-Interview Report)

面试结束后立即生成一份堪比专业猎头的分析报告。

1. **全景复盘报告**：
* **文字稿对照**：逐字稿展示，高亮显示答得好的金句和逻辑混乱的灾难区。
* **硬核评分板**：从业务理解、技术/学术深度、沟通表达、抗压能力四大维度打分。
* **视觉与语态分析**：基于录制的视频，指出用户的减分小动作（如频繁看天、语速过快、口头禅“然后”过多）。


2. **后续建议与职业规划**：
* **短期补救**：“针对本次暴露出的 X 问题，建议你在面试前恶补这 3 篇文献 / 这个开源项目的底层逻辑。”
* **长期规划**：基于用户的整体表现和前期性格倾向，给出职业发展路径的宏观建议。

## 五、 知识库负载与处理机制设计 (RAG Data Strategy)

由于您手头有大量结构化/非结构化的专业资料（HR 提问手册、大厂面经、保研经验），系统将这样处理这些数据：

1. **向量化切片 (Chunking)**：将《HR提问手册》按“考察维度（如：自驱力、领导力）”切片；将《大厂面经》按“公司名称-岗位-业务线”切片。
2. **混合检索 (Hybrid Search)**：当生成“真实模拟面试”的问题时，大模型会先提取用户的 JD 关键词，去向量库中检索对应的“真实历史问题”和“HR 考察标准模板”。
3. **System Prompt 注入**：将检索到的结果作为前置上下文喂给 AI，限制 AI 的人设。例如：*“你现在是面试官。考生的简历是[简历数据]，他应聘的是[岗位]。根据内部 HR 手册，该岗位重点考察[检索到的核心维度]。请结合面经中的真实案例[检索到的面经]，对他发起一个具有挑战性的提问。”*

这份设计将“要点提示”和“深挖追问”作为核心壁垒，能够完美解决顶尖本科生“背答案无用、缺乏真实对抗感”的痛点。您可以基于此骨架进入原型的敏捷开发阶段。
